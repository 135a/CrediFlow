# CrediFlow 智能体（Agent）架构指引

本文档面向全体研发团队，同步系统最新的微服务到“智能体”架构的演进边界与职责规范。

## 1. 核心智能体划分
系统抛弃了按技术组件切分单体的大泥球模式，将信贷全生命周期重塑为以下 5 大核心 Agent：

1. **用户服务 Agent (User Service)**
   - **技术栈**: Java (Spring Cloud)
   - **职责**: 进件引导、用户资料与资质收集、权限控制、消息推送。
   - **特征**: 系统的唯一前端入口之一，重交互。

2. **反欺诈与风控 Agent (Anti-Fraud & Risk)**
   - **技术栈**: Python
   - **职责**: 身份冒用检测、多头借贷识别、征信评分卡匹配、自动审批流。
   - **特征**: 强依赖于规则引擎与模型，提供 REST 接口供 Java 侧请求。

3. **额度定价 Agent (Credit Limit & Pricing)**
   - **技术栈**: Python
   - **职责**: 基于用户评级的“千人千额、千人千率”自动计算，确保合规费率不越界。
   - **特征**: 算力密集型，强合规校验。

4. **合约流程 Agent (Contract Process)**
   - **技术栈**: Java (Spring Cloud)
   - **职责**: 合同（借款协议、担保协议）渲染、状态机维护（待签署 -> 已签署）、外部电子签回调处理。
   - **特征**: 重状态流转与外部三方（签章）集成。

5. **贷后管理 Agent (Post-Loan Management)**
   - **技术栈**: Java (Spring Cloud)
   - **职责**: 还款计划拆分、自动代扣轮询、逾期判定与展期处理。
   - **特征**: 定时任务密集，强资金账务属性。

## 2. 跨域通信与一致性约定
1. **统一网关路由**: Java 与 Python 之间不允许硬编码 IP 进行直连点对点 RPC，必须经由全局 APISIX / Go 网关代理流转，并在头部统一传递 `X-Trace-Id` 与 `X-Agent-Source`。
2. **最终一致性**: 跨 Agent 的长业务链路（如：签约完成触发借据生成）须采用基于 MQ 的可靠投递或定时对账重试补偿。长流程中断抛出 `AgentAsyncRetryException` 异常体系处理。
