# 端到端联调测试清单 (12.4)

## 核心借款链路回归测试

| 模块 | 步骤说明 | 预期结果 | 状态 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **注册/登录** | 用户通过 App 发起注册并登录获取 JWT | 返回合法 JWT Token，包含 `userId` | [ ] | 网关校验通过 |
| **风控授信** | 调用 `/api/app/credit/apply` 发起授信 | 返回基础额度，生成 `cf_credit_result` 记录 | [ ] | Agent 超时自动降级 |
| **借款申请** | 调用 `/api/app/loan/apply` 提交申请 | 状态流转为 `PENDING` -> `APPROVED` | [ ] | 验证幂等 Token |
| **合同签署** | 前端勾选“阅读并同意”，请求 `/api/app/contract/sign` | 返回 PDF 下载链接，后台落库存证 | [ ] | PDF 包含印章占位 |
| **核心放款** | 模拟审批放款触发 | 生成借据，并调 `fund-flow-service` 记录放款流水 | [ ] | - |
| **计划生成** | 根据借据回调 `repayment-service` | 成功批量生成 `cf_repayment_plan` 数据 | [ ] | 状态均为 `PENDING` |
| **主动还款** | 用户选择当期计划，发起还款支付 | 调起收银台，流转至支付回调 | [ ] | 幂等锁拦截重复请求 |
| **回调对账** | 第三方支付回调 `/api/callback/payment` | 更新计划为 `PAID`，记录还款流水 | [ ] | 签名校验通过 |
| **逾期调度** | `batch-service` 模拟触发逾期巡检任务 | 计算罚息，更新计划为 `OVERDUE`，并生成催收任务 | [ ] | 查看调度日志 |
