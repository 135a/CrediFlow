## 背景与原因

当前 Go `batch-service` 仅实现了代扣和逾期巡检两个最简 Job，总代码不到 100 行，与设计文档中规划的"分布式任务调度中枢"定位严重不符。需要补全全部 6 类定时任务、引入 Redis 分布式锁、增加事件驱动能力和管理接口，使其真正成为系统的"任务大脑"。

## 变更内容

- 引入 `go-redis` 实现分布式锁，确保多实例部署安全
- 新增罚息计算、还款提醒、风控异步分发、消息推送调度 4 个 Job
- 增强现有代扣和逾期 Job（分页批量 + 并发 + 重试）
- 新增 HTTP 管理接口（健康检查、手动触发任务、查看调度状态）
- 新增任务执行日志上报模块

## 功能特性

### 新增功能特性
- `scheduler-go`: Go 分布式任务调度服务完整实现

### 修改的功能特性
_无_

## 影响范围

- **受影响代码**: `batch-service/` 下全部文件
- **新增依赖**: `go-redis/v9`
- **配置变更**: `.env` 新增 Redis 连接和各 Job 的 Cron/开关配置
- **Docker**: `docker-compose.yml` 中 `batch-service` 需增加 Redis 依赖
