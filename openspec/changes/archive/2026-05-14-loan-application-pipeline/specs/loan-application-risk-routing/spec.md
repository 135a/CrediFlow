## ADDED Requirements

### Requirement: 借款实时行为拦截与校验
系统 MUST 在接收到借款申请时实时捕获并校验用户的上下文与环境特征，针对异常环境直接阻断借款。

#### Scenario: 深夜高频发起借款
- **WHEN** 用户在凌晨 02:00 发起借款申请，且这已经是当晚的第 3 次尝试
- **THEN** 系统 MUST 拦截该申请并返回“系统繁忙”保护性话术，同时触发安全埋点记录

#### Scenario: 存在未结清在途借据
- **WHEN** 用户再次发起借款，但当前已有一笔逾期未还的在途借款
- **THEN** 系统 MUST 拦截申请，并提示“存在未结清款项，暂时无法继续借款”

### Requirement: 借款风控分层与流转
系统 MUST 建立明确的借款级矩阵路由。借款的风控评分计算应当调用现有的四维模型，但采取比授信更为严格的分层阈值判定。

#### Scenario: 低风险用户借款
- **WHEN** 用户的风险等级计算为 LOW
- **THEN** 系统 MUST 直接进入放款计划生成阶段，不触发二次人脸与人工审核

#### Scenario: 中风险用户借款
- **WHEN** 用户的风险等级计算为 MEDIUM
- **THEN** 系统 MUST 暂停流程，触发要求二次活体人脸验证，人脸验证成功后方可放款

#### Scenario: 高风险用户借款
- **WHEN** 用户的风险等级计算为 HIGH
- **THEN** 系统 MUST 要求二次人脸验证，且验证成功后 MUST 流转入 `PENDING_MANUAL_REVIEW` 人工审核队列
