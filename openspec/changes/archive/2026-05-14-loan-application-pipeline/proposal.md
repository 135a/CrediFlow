## Why

随着授信全流程的落地，用户已能成功获取循环可用额度并签署授信合约。此时，为了让产品能够真正进入信贷生命周期的资金流转阶段，亟需打通“从用户发起单笔借款申请到最终放款”的借款全链路（Loan Application Pipeline）。
与授信环节相比，借款直接关乎资金流出，因此需要引入更为严格的风控阈值、增加实时行为监控（如深夜操作、异地登录、高频申请）以及对应的分层路由拦截机制，并在审核通过后实时拆分生成还款计划并冻结对应的额度，从而保障资金链的安全与闭环。

## What Changes

- **实现借款风控实时行为校验**：在发起借款申请时实时校验 IP 属地、设备指纹、申请频次、深夜高危时段以及在途借款数量。
- **构建借款风控分层与流转机制**：建立类似授信的路由矩阵，但阈值更严。低风险直过；中风险必须二次人脸校验；高风险必须二次人脸+转人工，并呼叫 Agent 提供借款级别的三件套辅助审核。
- **生成借据与还款计划拆分**：审核通过后生成单笔独立的借据编号，并依据约定的期数和利率拆分出每期的还款计划明细（本金 + 利息）。
- **可用额度实时冻结机制**：生成借据后，需要同步去 `cf_user_credit_quota` 扣减/冻结用户对应的可用额度。

## Capabilities

### New Capabilities

- `loan-application-risk-routing`: 借款申请的实时行为校验（IP、设备、深夜操作、频次）以及风险分层确定性路由机制。
- `loan-application-manual-review`: 借款环节的高危人工审核以及 Agent 借款级别的风险评估三件套集成。
- `loan-contract-repayment-plan`: 借款通过后的单笔借据生成逻辑及等额本息/先息后本的还款计划拆分。

### Modified Capabilities

- `credit-quota-revolving`: 循环额度中心需要新增针对单笔借款成功后的“可用额度实时冻结与扣减”能力。

## Impact

- `loan-application-service`: 新增借款申请的全链路状态机流转。
- `credit-risk-service`: 增加独立的借款风控引擎规则及借款人工审核队列。
- `microservice-repayment` (暂未实现或待实现): 新增或承接借据和分期还款计划的落地逻辑。
- `credit-risk-service` 循环额度模块需要支持实时的并发冻结事务。
- `agent-python`: 现有的 `manual_review_assistant` 需要感知这是借款场景并提供对应的洞察和建议。
