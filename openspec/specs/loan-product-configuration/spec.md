## ADDED Requirements

### Requirement: 借款分期期数白名单配置
系统 MUST 在核心借款入口验证分期期数是否在支持的白名单范围内，以防止非标准的资产产生。支持的期数枚举 MUST 仅为 `3`、`6`、`12`。

#### Scenario: 用户传入合法的期数
- **WHEN** 客户端传入 `term=6` 发起借款申请
- **THEN** 系统 MUST 认为该期数合法并继续向下流转审核流程

#### Scenario: 用户传入非法的期数
- **WHEN** 恶意客户端或错误配置的终端传入 `term=4` 或 `term=24` 发起借款申请
- **THEN** 系统 MUST 强行拦截该请求，返回类似“不支持的分期期数”错误，并 MUST NOT 创建借款单

### Requirement: 全局借款与罚息费率配置读取
借据的年化利率与逾期罚息日利率 MUST 从环境配置中动态获取，以支持不同环境或业务线的随时调整。系统 MUST 保证当配置项缺失时能平滑降级到预设的默认值。

#### Scenario: 配置读取与降级
- **WHEN** 系统启动或生成借据/计算罚息时尝试读取费率配置
- **THEN** 系统 MUST 优先读取配置项（如年化利率配置），若该配置项未定义，则 MUST 回退到默认值 `0.18`（年化 18%）和 `0.0005`（罚息日利 万分之五）
