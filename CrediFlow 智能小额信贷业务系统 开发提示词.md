# CrediFlow 智能小额信贷业务系统 开发提示词

项目名称：CrediFlow 智能小额信贷业务系统

项目定位：面向小额信贷场景的企业级后端综合项目，融合微服务架构、云原生网关、分布式任务调度与AI智能决策能力，聚焦金融业务合规落地与智能化升级，适配校招简历展示与工业级工程实践，不涉及前端页面开发，专注后端架构与业务实现。

整体技术架构：SpringCloud Alibaba（微服务基座） \+ Apache APISIX（开源Go网关） \+ Go 分布式任务调度 \+ Python AI Data Agent（智能大脑）

## 一、架构整体设计

采用分层微服务架构，严格遵循“高内聚、低耦合”原则，整体架构链路清晰、可扩展、可观测，适配金融系统高可用、高安全、高合规要求，架构层级（从接入到存储）如下：

逻辑接入层（App用户端、Web后台管理端，仅做角色与接口适配，不开发实际前端页面） → Apache APISIX 开源云原生网关 → SpringCloud Alibaba 微服务集群 → Go 分布式任务调度服务 → Python 智能Data Agent服务 → 数据存储层。

服务间通信策略：同步通信采用 HTTP/HTTPS 接口（基于OpenFeign实现服务间远程调用），异步通信采用 RocketMQ 消息队列，实现解耦、削峰填谷，保障金融业务数据一致性与系统稳定性；整体采用 Docker \&amp; Docker Compose 容器化编排部署，支持一键部署、弹性扩容，降低环境部署成本，适配云原生落地场景。

额外补充：架构设计预留可观测性接口，支持集成 Prometheus \+ Grafana 实现系统监控、链路追踪，便于问题排查与性能调优，贴合企业级生产落地标准。

## 二、各技术栈分工（职责清晰、边界明确，不重复造轮子）

### 1\. Java 端：SpringCloud Alibaba 微服务架构（核心业务基座）

核心定位：承载小额信贷全流程确定性核心金融业务，保障业务合规、数据安全与事务一致性，是整个系统的业务基石。

技术组件（最简够用、不冗余）：Nacos（注册中心\+配置中心，实现服务注册发现、配置动态刷新）、OpenFeign（微服务间远程调用，简化接口调用逻辑）、Sentinel（熔断限流、降级保护，防止服务雪崩，保障高并发场景稳定）、MyBatis\-Plus（ORM框架，简化数据库操作）、MySQL（核心业务数据存储）、Redis（缓存热点数据、实现分布式锁，提升系统性能）、RocketMQ（异步消息通信，解耦服务）、ShardingSphere（可选，实现分库分表，应对数据量增长）、Spring Security（配合JWT，实现服务级权限管控）。

按业务领域严格拆分微服务（职责单一、边界清晰），各服务独立部署、独立迭代：

- 用户服务（user\-service）：用户注册、登录、短信/人脸认证、个人信息管理、用户画像采集，负责用户全生命周期管理，实现数据脱敏（身份证、手机号等敏感信息加密存储）。

- 授信风控服务（credit\-risk\-service）：授信额度评估、风控规则校验、用户风险等级判定，对接Python AI Agent获取智能风控建议，输出最终授信结果。

- 贷款申请服务（loan\-application\-service）：借款申请提交、资料审核、申请状态流转、放款资格校验，联动授信服务与合同服务，保障申请流程合规。

- 借款合同服务（loan\-contract\-service）：电子合同生成、签署、归档、查询，实现合同合规校验，留存操作日志，满足金融监管要求。

- 还款分期服务（repayment\-service）：还款计划生成、主动还款、分期申请、还款金额计算（本金\+利息\+手续费），支持多种还款方式适配。

- 贷后逾期管理服务（post\-loan\-service）：逾期判定、罚息计算、逾期状态更新、催收任务发起，联动Go调度服务与Python Agent实现风险预警。

- 资金流水对账服务（fund\-flow\-service）：放款/还款资金流水记录、对账校验、流水统计，保障资金账务清晰，支持与第三方支付接口预留对接。

- 系统权限后台服务（system\-service）：角色管理、菜单管理、权限分配、操作日志审计，实现后台运营、风控、管理员的权限隔离，保障系统安全。

Java端核心职责：专注于确定性金融业务逻辑实现，重点保障事务一致性、数据脱敏、幂等设计（防止重复放款、重复还款）、操作日志可追溯，严格遵循金融合规要求，不涉及AI智能决策与高并发调度。

### 2\. 网关层：Apache APISIX 开源云原生网关（全局流量入口）

核心定位：统一流量管控、安全防护、路由转发，替代自研网关，复用企业级成熟组件，降低开发成本与安全风险，适配金融系统高安全要求。

核心功能：路由转发（将不同端、不同业务的请求转发至对应微服务）、JWT统一鉴权（全局身份校验，区分普通用户与管理员权限）、限流熔断（限制单用户/单接口请求频率，防止恶意请求与服务过载）、IP黑白名单（拦截恶意IP，保障系统安全）、请求日志采集（记录所有入口请求，便于审计与问题排查）、灰度路由（支持服务版本灰度发布，降低迭代风险）、安全防护（拦截SQL注入、XSS等恶意攻击，适配金融级安全标准）。

设计原则：不承担业务逻辑，仅做流量层管控，与微服务、调度服务、AI Agent解耦，保障网关高可用、低延迟。

### 3\. Go 端：分布式任务调度服务（高并发任务中枢）

核心定位：专注于分布式定时任务、异步任务的调度与执行，利用Go语言高并发、低内存、编译部署简单的特性，保障任务高吞吐、低延迟，适配金融场景下定时任务的准确性与可靠性。

核心任务：自动代扣还款（按还款计划定时发起代扣请求，联动还款服务）、逾期巡检（定时扫描逾期订单，更新逾期状态，发起催收任务）、罚息计算（定时计算逾期罚息，同步至还款服务与资金流水服务）、到期还款提醒（定时推送还款提醒消息，联动消息服务）、风控异步任务分发（将耗时风控任务异步分发至Python Agent，避免阻塞核心业务）、消息推送调度（统一调度用户通知、逾期提醒等消息，保障推送及时性）。

技术亮点：实现任务分片、失败重试、任务监控，支持动态调整任务频率，保障定时任务不遗漏、不重复执行，贴合金融场景下任务调度的高可靠性要求。

### 4\. Python 端：金融一体化 Data Agent（系统智能大脑）

核心定位：独立部署为服务，承担整个系统的AI智能决策、语义理解、数据分析能力，作为Java后端业务的智能补充，提升业务效率、降低人工成本，是系统差异化亮点。

技术组件：LangGraph（Agent流程编排，实现多步骤智能决策）、RAG（检索增强生成，构建金融信贷知识库）、向量库（存储金融知识库向量，提升检索效率）、NL2SQL（自然语言转SQL，实现业务数据快速查询）、NL2API（自然语言转API调用，实现业务接口便捷操作）、LLM（基础大模型，实现语义理解与智能推理）。

核心能力（贴合小额信贷场景，落地性强）：

- 智能风控辅助审核：对接授信风控服务，调取用户数据，结合金融风控知识库，智能分析用户风险等级，给出授信建议与审核意见，辅助人工审核，提升审核效率。

- 贷后风险智能预警：实时监控用户还款状态、逾期趋势，结合历史数据智能分析，提前识别坏账风险，推送预警信息至后台管理端，辅助贷后管控。

- NL2SQL 自然语言业务查数：支持运营、风控人员通过自然语言查询业务数据（如“查询本月放款总额”“统计逾期用户占比”），自动转换为SQL语句查询数据库，整理成结构化报表返回，无需手动编写SQL。

- NL2API 自然语言调用业务接口：支持通过自然语言发起业务操作（如“冻结某用户授信额度”“导出本月还款记录”），自动匹配对应Java微服务接口，完成参数组装与调用，简化操作流程。

- 金融知识库RAG智能咨询：构建小额信贷相关知识库（如利率规则、还款规则、逾期处罚规则），支持用户、运营人员通过自然语言咨询，自动给出准确、合规的回答，替代部分人工客服。

- 自动业务数据分析：定时生成日报、周报、月报，统计放款量、通过率、逾期率、坏账率等核心指标，智能分析指标变化原因，给出业务优化建议，辅助运营决策。

- 逾期趋势智能归因：针对逾期率升高、坏账增多等问题，智能分析背后原因（如用户画像、市场环境、风控规则等），输出归因报告，为风控规则优化提供支撑。

设计原则：独立于Java后端业务，通过HTTP接口与微服务、Go调度服务通信，不参与核心业务数据存储与事务控制，仅提供智能决策与分析能力，便于独立迭代与升级。

## 三、数据存储层设计（贴合金融数据安全要求）

1\. 关系型数据库：MySQL，存储核心业务数据（用户信息、授信记录、贷款订单、还款记录、资金流水等），重要表采用分库分表（ShardingSphere），保障数据存储扩展性；敏感数据采用加密存储，满足数据脱敏要求。

2\. 缓存数据库：Redis，存储热点数据（用户登录信息、授信额度、还款计划等），实现分布式锁，提升系统查询性能，减少数据库压力。

3\. 向量库：用于存储Python Agent的金融知识库向量数据，支撑RAG检索功能，提升语义检索效率与准确性。

4\. 消息队列：RocketMQ，存储异步消息（如放款通知、逾期提醒、任务调度消息等），实现服务解耦，保障消息可靠传输。
